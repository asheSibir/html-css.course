Помогают ли предлагаемые вами устройства достичь в загородном доме комфорта гороского уровня?

Нет, не может. Армированное изделие и марка бетона М-350, позволяют выдерживать значительные нагрузки, включая землетрясение до 5,5 баллов.

Погибают ли бактерии в холодный период при долгом простое системы?

Да. Иногда зимний монтаж предпочтительней. Большегрузная техника по мороженому грунту не продавит и не испортит дорогу или ландшафт участка, да и грунтовая вода отходит. Весной грунт даст усадку и уплотнится, и вам останется сделать лишь ландшафтное планирование.

JavaScript

Модульная структура и webpack

Создать папку src. В ней папка modules и index.js
В каждом модуле прописать export default "name of function"
В index.js прописать import name of function from './...'
Затем вызов функции name of function.()

Устанавить webpack (Подробнее - Get_Started ). Для этого ввести 2 команды в терминал :
npm init -y
 npm install webpack webpack-cli -D (npm install webpack webpack-cli --save-dev)

Чтобы остановить установку, нужно набрать CTRL+C
-D означает, что установка использоваться будет только для разработки, не для релиза
npm i --cache /tmp/empty-cache Чтобы почистить кэш ноды

Запустить сборку в dist через ввод в терминале npx webpack

GITIGNORE: создать файл .gitignore и прописать в нем /node_modules/

Устанавить wepback server npm install --save-dev webpack-dev-server. Подробнее: ТУТ

В package.json изменить раздел scripts. Таким образом настроить две модификации: разработка и релиз. Его новый вид:

"scripts": {
"dev": "webpack-dev-server --open --mode=development",
"build": "webpack --mode=production"
},

Создать файл webpack.config.js в корневой папке.

В консоли: npm i path -D/

В самом файле .webpack.config.js:
const path = require('path');
module.exports = {
entry: {
main: './src/index.js',
},
output: {
path: path.resolve(__dirname, './dist'),
filename: '[name].js',
publicPath: '/dist'
},
devServer: {
overlay: true
},
};

Запустить первый раз dev npm run dev

Установить babel, для этого ввести в терминале npm install -D babel-loader @babel/core @babel/preset-env. Это первая команда ТУТ

Дописать в webpack.config.js module: {
rules: [
{
test: /\.js$/,
loader: 'babel-loader',
exclude: '/node_modules/'
}
]
}

Создать файл .babelrc

В файле .babelrc прописать: { "presets": ["@babel/preset-env"] }

Добавить @babel/polyfill. Для этого запустить команду npm install --save @babel/polyfill
В файл index.js добавить полифил import '@babel/polyfill';

npm run build

Data-атрибуты

Методы работы с атрибутами:
hasAttribute('nameAttr') - проверяет наличие атрибута
getAttribute('nameAttr') - возвращает значение атрибута
removeAttribute('nameAttr') - удаляет атрибут
setAttribute('nameAttr', 'value') - настраивает значение атрибута
Element.dataset - возвращает все data атрибуты элемента
Element.dataset('new-att') - добавляет новый data-атрибут

Навешиваем слушателя на событие, результатом которого должно быть появление data атрибута
В обработчике присваиваем атрубуту элемента значение data-атрибута
Например:
event.target.src = event.target.dataset.img
